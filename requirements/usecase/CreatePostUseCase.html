Title: Create Post
Primary Actor: User
Stakeholders and Interests:

User - wants to create a new post/topic on the platform.

Precondition: User must be authenticated and logged in.
Trigger: User clicks the "New Post" button from the main top navigation bar.
Main success scenario:

  1. Verified users click the "New Post" button in the main top navigation bar.
  2. User types up intented post in the textbox.
  3. User selects available formatting tools, as desired.
  4. User selects a prefered discussion group to make post avialable to, with the disscusion group defaulting to "None".
  5. User clicks the "Post" botton to display/submit new post on the platform.
  6. Data is passed to the backend, where the post is created.

Extensions:

  1. 5a. User submits an empty post:
    1. 2a1. Empty post is rejected.
    2. 2a2. Returns to "New Post" page with a prompt stating the post cannot be empty.
