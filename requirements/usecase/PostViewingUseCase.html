Title: Post Viewing
Primary Actor: Author
Stakeholders and Interests:

Author - wants to see and access a post to edit.

Precondition: Author must be logged into the platform.
Trigger: Author must click the "edit post" button.
Main success scenario:

  1. Author must be on the post viewing page to see and access post.
  2. Author can click the "edit post" button to add or remove content.
  3. Autor is redirected to the "new post" page to edit the post.
  4. "New post" button initoatesCreate Post use case.
  5. Autor can access the formatting tools as desired for editing post.
  6. User selects a prefered discussion group to make post avialable to, with the disscusion group defaulting to "None".
  7. User clicks the "Post" botton to display/submit edited post on the platform.
  8. Data is passed to the backend, where the edited post is created.
  9. Non-Authors can access this post and rate it out of 5 points.

Extensions:

  1. 2a. User is not the Author of the post:
    1. 2a1. User is prompted on the post cannot be edited if it was not created by user.
    2. 2a2. User is redirected to the "new post" page.
    3. 2a3. New post initiates Create Post use case
