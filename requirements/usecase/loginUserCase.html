Title: Login Authentication
Primary Actor: User
Stakeholders and Interests:

User - wants to access the platform via a login form.

Precondition: User should have an already registered account.
Trigger: User click "login" button on login form.
Main success scenario:

  1. User inputs a valid and distinct username and password.
  2. User click the "login" button.
  3. The users data is passed to the backend and then granted access to the platform.
  4. User has full access to main platform to continue other actions.
  5. User repeats 1 & 2 everytime they want to login.

Extensions:

  1. 3a. User enters an unregistered unique/distinct username and password:
    1. 3a1. SuD does not recognize users information.
    2. 3a2. SuD rejects users information.
    3. 3a3. User is prompted to create/register a new account.
    4. 3a4. User clicks the "not registered" button to register new account.
      6. 3a5. "Not Registered" button initiates Registration use case.
